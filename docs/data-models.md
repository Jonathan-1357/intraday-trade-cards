# Data Models
## Intraday Trade Card System

---

## Database

SQLite, stored at `apps/api/trade_cards.db`.
Tables are created automatically on startup via `Base.metadata.create_all()`.
To reset: delete `trade_cards.db` and restart the API. Default rows are re-seeded.

Source: `apps/api/app/models/__init__.py`

---

## TradeCard

Represents a single trade suggestion generated by the signal evaluator.

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | string (UUID) | — | Primary key |
| `symbol` | string | — | NSE symbol, e.g. `ICICIBANK` |
| `action` | string | — | `buy` or `sell` |
| `entry` | float | — | Suggested entry price (₹) |
| `stop_loss` | float | — | ATR-based stop loss price (₹) |
| `target` | float | — | Target price at 2:1 R:R (₹) |
| `quantity` | integer | — | Shares: `floor(risk / abs(entry - sl))` |
| `confidence` | string | — | `none` / `weak` / `valid` / `strong` |
| `status` | string | `generated` | Lifecycle state (see below) |
| `reasons` | JSON array | `[]` | Passing signal rules |
| `risk_reward` | float | `0.0` | `abs(target - entry) / abs(entry - sl)` |
| `capital_required` | float | `0.0` | `entry × quantity` (₹) |
| `archived` | boolean | `false` | Hidden from dashboard when true |
| `created_at` | datetime | now | Row creation time |
| `updated_at` | datetime | now | Last modification time |

**Indexes:** `status`, `created_at`

### Card Status Lifecycle

```
generated → valid → waiting → triggered → active ─→ completed
                                                  └→ invalidated
```

| Status | Meaning |
|---|---|
| `generated` | Just created, not yet confirmed |
| `valid` | Setup looks good, not yet at entry |
| `waiting` | Entry price not yet reached |
| `triggered` | Price has crossed entry |
| `active` | Inside the trade range (between entry and target/SL) |
| `invalidated` | SL hit or setup broke before completion |
| `completed` | Target reached |

Terminal states (`invalidated`, `completed`) are never re-evaluated by refresh.

## RiskConfig

Single-row table (always `id = 1`). Controls quantity calculations for all cards.

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | integer | `1` | Primary key (singleton) |
| `total_capital` | float | `100000.0` | Total trading capital (₹) |
| `risk_per_trade` | float | `1000.0` | Max risk per trade (₹ or %) |
| `risk_mode` | string | `fixed` | `fixed` (₹ amount) or `percent` |
| `max_concurrent_trades` | integer | `5` | Max number of active positions |

Quantity formula:
```
quantity = floor(risk_per_trade / abs(entry - stop_loss))
```

## Watchlist

Single-row table (always `id = 1`). Stores the list of symbols evaluated on each generate call.

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | integer | `1` | Primary key (singleton) |
| `symbols` | JSON array | `["ICICIBANK", "RELIANCE", "INFY", "HDFCBANK", "TCS"]` | NSE symbols to evaluate |

## Mock Quote Fields

`get_mock_quote(symbol)` returns a dict used by the signal evaluator.
All fields are deterministically seeded by `hash(symbol)`.

### Core Price

| Field | Type | Range | Description |
|---|---|---|---|
| `symbol` | string | — | Symbol passed in |
| `price` | float | 500–3000 | Current price (₹) |
| `open` | float | ±1% of price | Session open price |
| `high` | float | price–price+2% | Intraday high |
| `low` | float | price–2%–price | Intraday low |
| `vwap` | float | low–high range | Volume-weighted average price |
| `prev_close` | float | ±2% of price | Previous session close |

### Momentum

| Field | Type | Range | Description |
|---|---|---|---|
| `rsi` | float | 30–75 | Relative Strength Index |
| `stoch_rsi` | float | 0–100 | Stochastic RSI |
| `macd_histogram` | float | varies | MACD histogram value (+ = bullish) |

### Volume

| Field | Type | Range | Description |
|---|---|---|---|
| `volume` | integer | 100k–5M | Today's volume |
| `avg_volume` | integer | 80k–3M | 20-day average volume |
| `volume_ratio` | float | derived | `volume / avg_volume` |

### Trend

| Field | Type | Values | Description |
|---|---|---|---|
| `ema9` | float | near price | 9-period EMA |
| `ema21` | float | near price | 21-period EMA |
| `ema50` | float | near price | 50-period EMA |
| `obv_direction` | string | `up/down/flat` | On-Balance Volume trend |

### Volatility

| Field | Type | Range | Description |
|---|---|---|---|
| `atr` | float | 0.5–2% of price | Average True Range (₹) |
| `bb_width` | float | 0.01–0.05 | Bollinger Band width as fraction of price |

### Structure

| Field | Type | Values | Description |
|---|---|---|---|
| `gap_pct` | float | derived | Gap from prev close: `(open - prev_close) / prev_close` |
| `or_high` | float | derived | Opening range high (first 15 min) |
| `or_low` | float | derived | Opening range low (first 15 min) |
| `or_breakout` | string | `up/down/none` | Whether price broke out of the opening range |

### Market Context

| Field | Type | Values | Description |
|---|---|---|---|
| `nifty_direction` | string | `up/down/flat` | Nifty 50 direction (same for all symbols) |
| `sector_direction` | string | `up/down/flat` | Sector direction for this symbol |

### Filters

| Field | Type | Description |
|---|---|---|
| `near_circuit` | boolean | `True` if price is within 2% of ±20% circuit |
| `in_time_window` | boolean | `True` if current time is in valid trading window (always `True` in mock) |

## Shared JSON Schema

The canonical trade card shape is defined at:
`shared/trade-card-schema/trade-card.json`

This schema matches the `TradeCard` database model and is the source of truth for both
the FastAPI Pydantic schemas (`apps/api/app/schemas/__init__.py`) and the
TypeScript types (`apps/web/types/index.ts`).
