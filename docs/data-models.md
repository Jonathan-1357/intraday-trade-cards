# Data Models
## IRIS EDGE — Intraday Trade Card System

All models use SQLAlchemy ORM with a SQLite backend (`trade_cards.db`).
Tables are created automatically via `Base.metadata.create_all()` on startup.

---

## TradeCardModel

Table: `trade_cards`

| Column | Type | Description |
|---|---|---|
| `id` | String (UUID) | Primary key |
| `symbol` | String | NSE trading symbol |
| `action` | String | `buy` or `sell` |
| `entry` | Float | Suggested entry price |
| `stop_loss` | Float | ATR-based stop level |
| `target` | Float | 2× risk projection |
| `quantity` | Integer | Shares to trade |
| `confidence` | String | `none` \| `weak` \| `valid` \| `strong` |
| `status` | String | Lifecycle status (see below) |
| `reasons` | JSON | Array of human-readable rule strings |
| `risk_reward` | Float | Risk/reward ratio |
| `capital_required` | Float | `entry × quantity` |
| `archived` | Boolean | Soft-delete flag (default `false`) |
| `created_at` | DateTime | Server-set on insert |
| `updated_at` | DateTime | Server-set on update |

**Indexes**: `status`, `created_at`

### Status values

| Status | Meaning |
|---|---|
| `generated` | Initial state (legacy — use `valid`) |
| `valid` | Ready-to-trade card |
| `pre_open` | Generated before market open (buy-only) |
| `waiting` | Monitoring for entry price hit |
| `triggered` | Entry price reached |
| `active` | Position open |
| `completed` | Target hit — profitable close |
| `invalidated` | Stop loss hit — loss close |

Terminal states (no further transitions): `completed`, `invalidated`

## RiskConfigModel

Table: `risk_config`

Single row (always `id = 1`). Seeded on startup if not present.

| Column | Type | Default | Description |
|---|---|---|---|
| `id` | Integer | 1 | Always 1 |
| `total_capital` | Float | 100,000 | Total trading capital in ₹ |
| `risk_per_trade` | Float | 1,000 | Risk amount per trade |
| `risk_mode` | String | `fixed` | `fixed` (₹) or `percent` (%) |
| `max_concurrent_trades` | Integer | 5 | Maximum simultaneous open positions |

---

## WatchlistModel

Table: `watchlist`

Single row (always `id = 1`). Seeded on startup.

| Column | Type | Default |
|---|---|---|
| `id` | Integer | 1 |
| `symbols` | JSON (array) | `["ICICIBANK", "RELIANCE", "INFY", "HDFCBANK", "TCS"]` |

---

## ApiConfigModel

Table: `api_config`

Single row. Persists the Upstox access token across server restarts.

| Column | Type | Description |
|---|---|---|
| `id` | Integer | Always 1 |
| `upstox_token` | String | JWT access token (empty string if not set) |

On startup, `load_token_from_db()` reads this row and sets `settings.upstox_access_token` in-memory so all routes have immediate access.

---

## DailyWatchlistModel

Table: `daily_watchlist`

One row per symbol per day (generated by the auto-scan engine).

| Column | Type | Description |
|---|---|---|
| `id` | String (UUID) | Primary key |
| `date` | String | `YYYY-MM-DD` |
| `symbol` | String | NSE trading symbol |
| `score` | Integer | Composite score (0–100+) |
| `rank` | Integer | Rank within today's entries (1 = highest score) |
| `category` | String | Grouping label (see below) |
| `action` | String | `buy` or `sell` |
| `reason_tags` | JSON | Array of tag strings (e.g. `gap_up`, `volume_spike`) |
| `indicator_snapshot` | JSON | Dict of key indicator values at time of scoring |
| `dismissed` | Boolean | User dismissed this entry |
| `traded` | Boolean | User marked as traded |
| `created_at` | DateTime | When the entry was generated |

**Index**: `date`

### Category values
- `Gap Up Momentum`
- `Gap Down Reversal`
- `Breakout Candidate`
- `High Volume Mover`
- `Strong Momentum`

---

## PaperWalletModel

Table: `paper_wallet`

Single row. Tracks paper trading virtual balance.

| Column | Type | Default | Description |
|---|---|---|---|
| `id` | Integer | 1 | Always 1 |
| `enabled` | Boolean | `false` | Whether paper mode is active |
| `balance` | Float | 0.0 | Current available paper balance |
| `initial_balance` | Float | 0.0 | Starting balance (set when first funded) |

`balance` decrements on buy, increments on sell/close.
`initial_balance` is set when `set_balance` is called and never changes unless reset again.

---

## PaperPositionModel

Table: `paper_positions`

One row per open paper position (per symbol per direction).

| Column | Type | Description |
|---|---|---|
| `id` | String (UUID) | Primary key |
| `symbol` | String | NSE trading symbol |
| `action` | String | `buy` or `sell` (direction of position) |
| `quantity` | Integer | Number of shares held |
| `avg_price` | Float | Volume-weighted average fill price |
| `created_at` | DateTime | When the position was opened |

**Index**: `symbol`

Partial fills are averaged into the existing row. When quantity reaches 0, the row is deleted.

---

## PaperOrderModel

Table: `paper_orders`

One row per paper order placed. Retained indefinitely (last 50 returned via API).

| Column | Type | Description |
|---|---|---|
| `id` | String (UUID) | Primary key |
| `symbol` | String | NSE trading symbol |
| `action` | String | `buy` or `sell` |
| `order_type` | String | `LIMIT` \| `MARKET` \| `SL` \| `SL-M` |
| `quantity` | Integer | Shares ordered |
| `price` | Float | Requested price (0 for MARKET) |
| `executed_price` | Float | Actual fill price (LTP at time of order) |
| `status` | String | Always `complete` (paper orders fill immediately) |
| `created_at` | DateTime | Order timestamp |

**Index**: `symbol`

## Pydantic Schemas (API I/O)

Defined in `schemas/__init__.py`. All use `model_config = ConfigDict(from_attributes=True)`.

| Schema | Used for |
|---|---|
| `TradeCardSchema` | Reading/returning trade card data |
| `RiskConfigSchema` | Reading/updating risk config |
| `WatchlistSchema` | Reading/updating watchlist |

## TypeScript Types (Frontend)

Defined in `types/index.ts`.

```typescript
type Action = "buy" | "sell"
type Confidence = "none" | "weak" | "valid" | "strong"
type CardStatus = "generated" | "valid" | "pre_open" | "waiting" | "triggered" | "active" | "invalidated" | "completed"

interface TradeCard {
  id: string
  symbol: string
  action: Action
  entry: number
  stop_loss: number
  target: number
  quantity: number
  confidence: Confidence
  status: CardStatus
  reasons: string[]
  risk_reward: number
  capital_required: number
  archived: boolean
  created_at: string
  updated_at: string
}

interface RiskConfig {
  id: number
  total_capital: number
  risk_per_trade: number
  risk_mode: "fixed" | "percent"
  max_concurrent_trades: number
}
```
